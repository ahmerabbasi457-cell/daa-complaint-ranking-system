SECTION D

Algorithm Information

Core techniques powering this system

Min-Heap · Top-K Optimization
Priority queue maintains K highest-scored complaints. Root = minimum of Top-K — enables O(log K) insert & O(1) min-access.
O(n log k)
Time Decay Ranking
Score × e−λt — Older complaints decay exponentially, surfacing fresh issues dynamically without manual curation.
e⁻λt
Spam Detection
Text similarity via Jaccard/cosine distance flags duplicate & near-duplicate complaints, preventing score inflation from coordinated spam.
Filter
Clustering
K-Means / hierarchical clustering groups related complaints into topics, enabling cluster-weighted scoring and category-level analytics.
Groups
Greedy Approximation
At each insert: if score > heap root → evict min, insert new. O(log K) per operation. Proves optimal for Top-K selection without full sort.
Optimal
Top-K via Heap O(n log k)
vs Full Sort O(n log n)
Space O(k)
Per-insert O(log k)
